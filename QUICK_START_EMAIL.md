✅ **Database migration** for the new field  
✅ **Dependency injection** in Program.cs  

## Configuration Required

Update `appsettings.json` with your email provider credentials:

```json
"Email": {
  "SmtpServer": "smtp.gmail.com",
  "SmtpPort": "587",
  "SenderEmail": "your-email@gmail.com",
  "SenderPassword": "your-app-password",
  "SenderName": "On-Screen Marking"
}
```

## For Gmail Users

1. Go to: https://myaccount.google.com/apppasswords
2. Select "Mail" and "Windows Computer"
3. Copy the 16-character password
4. Paste it as `SenderPassword` in appsettings.json
5. Use your Gmail address as `SenderEmail`

## How It Works

### Registration Flow:
```
User submits registration
    ↓
API generates 12-char random password
    ↓
User record created with auto-generated password
    ↓
Email sent with credentials
    ↓
User can login with either password
```

### Email Content:
- Professional HTML format
- Contains email and auto-generated password
- Security reminder to change password
- Branded footer

## Apply Migration

Run this command to update the database:

```bash
dotnet ef database update
```

## Testing

1. Update `appsettings.json` with valid SMTP credentials
2. Run: `dotnet run`
3. Call POST `/api/auth/register` with test data
4. Check email for auto-generated password

## Error Handling

- Email failures don't block registration
- Errors are logged to console
- System continues to work even if email service is down

## Security Notes

- Auto passwords: 12 chars with uppercase, lowercase, numbers, special chars
- Passwords hashed with SHA256 before storage
- SMTP uses SSL/TLS encryption
- Store credentials securely in production (use environment variables)
