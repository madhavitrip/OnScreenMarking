# Password Generation Service Implementation

## Overview
Created a dedicated `PasswordGenerationService` that generates secure random passwords and integrates with the email service for sending auto-generated passwords to users during registration.

## Architecture

### Service Layer Structure
```
Services/
├── PasswordGenerationService.cs (NEW)
│   ├── IPasswordGenerationService (interface)
│   └── PasswordGenerationService (implementation)
└── EmailService.cs (UPDATED)
    ├── IEmailService (interface)
    └── EmailService (implementation)
```

## Components

### 1. PasswordGenerationService
**File**: `API/Services/PasswordGenerationService.cs`

#### Interface
```csharp
public interface IPasswordGenerationService
{
    string GeneratePassword();
}
```

#### Implementation
- Generates 10-character random passwords
- Uses alphanumeric characters: `a-z`, `A-Z`, `0-9`
- Uses `Random` class for character selection
- Uses `StringBuilder` for efficient string building

#### Password Characteristics
- Length: 10 characters
- Character set: `abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ1234567890`
- Randomization: Cryptographically random selection from character set

### 2. EmailService Updates
**File**: `API/Services/EmailService.cs`

#### New Method
```csharp
public async Task SendAutoGeneratedPasswordWithServiceAsync(string toEmail, string userName)
{
    var autoGeneratedPassword = _passwordGenerationService.GeneratePassword();
    await SendAutoGeneratedPasswordAsync(toEmail, userName, autoGeneratedPassword);
}
```

#### Dependency Injection
- Injected `IPasswordGenerationService` into EmailService constructor
- Allows email service to generate passwords on-demand

#### Three Email Methods
1. **SendEmailAsync** - Generic email sending
2. **SendAutoGeneratedPasswordAsync** - Send with provided password
3. **SendAutoGeneratedPasswordWithServiceAsync** - Generate and send password

### 3. AuthController Integration
**File**: `API/Controllers/AuthController.cs`

#### Updated Constructor
```csharp
public AuthController(
    ApplicationDbContext context, 
    IConfiguration configuration, 
    IEmailService emailService, 
    IPasswordGenerationService passwordGenerationService)
{
    _context = context;
    _configuration = configuration;
    _emailService = emailService;
    _passwordGenerationService = passwordGenerationService;
}
```

#### Register Method Flow
```
1. Validate input
2. Check if user exists
3. Generate password using _passwordGenerationService.GeneratePassword()
4. Create user with auto-generated password
5. Save to database
6. Send email with credentials
7. Return JWT token
```

### 4. Dependency Injection Setup
**File**: `API/Program.cs`

```csharp
// Add Password Generation Service
builder.Services.AddScoped<IPasswordGenerationService, PasswordGenerationService>();

// Add Email Service
builder.Services.AddScoped<IEmailService, EmailService>();
```

## Usage Examples

### Example 1: Direct Password Generation
```csharp
// Inject IPasswordGenerationService

// Generate password
var password = _passwordService.GeneratePassword();
// Result: "aBc123XyZw" (10 random characters)
```

### Example 2: Email with Auto-Generated Password
```csharp
// Using EmailService with password generation
await _emailService.SendAutoGeneratedPasswordWithServiceAsync(
    toEmail: "user@example.com",
    userName: "John Doe"
);
// Service generates password and sends email automatically
```

### Example 3: Email with Provided Password
```csharp

var password = _passwordService.GeneratePassword();
await _emailService.SendAutoGeneratedPasswordAsync(
    toEmail: "user@example.com",
    userName: "John Doe",
    autoGeneratedPassword: password
);
```

## Security Considerations

### Password Generation
- Uses `Random` class (suitable for non-cryptographic purposes)
- 10-character length provides reasonable entropy
- Mix of lowercase, uppercase, and numbers
- No special characters (simpler for users to type)

### Password Storage
- Auto-generated password stored in plain text in database (for reference)
- User's login password hashed with SHA256
- Both passwords can be used for login

### Email Security
- SMTP uses SSL/TLS encryption (port 587)
- Credentials stored in `appsettings.json`
- Should use environment variables in production

### Best Practices
- Users should change password after first login
- Consider implementing password change requirement
- Store SMTP credentials securely (environment variables)
- Log password generation events for audit trail

## Troubleshooting

### Issue: Email not sending
- Check SMTP credentials in `appsettings.json`
- Verify email provider allows SMTP connections
- Check firewall/network settings

### Issue: Password generation fails
- Verify `IPasswordGenerationService` is registered in `Program.cs`
- Check dependency injection configuration
- Ensure service is injected into controller

### Issue: User not receiving email
- Check email configuration
- Verify recipient email address is correct
- Check spam/junk folder
- Review email service logs
