## 🏗️ Architecture

### Service Hierarchy
```
PasswordGenerationService
    ↓ (injected into)
EmailService
    ↓ (injected into)
AuthController
```

### Dependency Injection Flow
```
Program.cs
├── Register IPasswordGenerationService
├── Register IEmailService
└── Inject into AuthController

AuthController
├── Uses IPasswordGenerationService to generate password
├── Uses IEmailService to send email
└── Stores password in User model
```

## 🔄 Registration Flow

```
1. User submits registration request
   ↓
2. AuthController validates input
   ↓
3. Generate password using PasswordGenerationService
   ↓
4. Create User object with auto-generated password
   ↓
5. Save user to database
   ↓
6. Send email using EmailService with password
   ↓
7. Return JWT token to user
```

## 📧 Email Integration

### Three Email Methods Available

1. **SendEmailAsync()**
   - Generic email sending
   - Parameters: toEmail, subject, body
   - Usage: Any email notification

2. **SendAutoGeneratedPasswordAsync()**
   - Send email with provided password
   - Parameters: toEmail, userName, autoGeneratedPassword
   - Usage: When password is already generated

3. **SendAutoGeneratedPasswordWithServiceAsync()**
   - Generate password and send email
   - Parameters: toEmail, userName
   - Usage: One-step password generation and email

## 🔐 Security Features

### Password Generation
- ✅ 10-character length
- ✅ Alphanumeric characters (a-z, A-Z, 0-9)
- ✅ Random selection from character set
- ✅ No special characters (easier for users)

### Password Storage
- ✅ Auto-generated password stored in database
- ✅ User's login password hashed with SHA256
- ✅ Both passwords can be used for login

### Email Security
- ✅ SMTP uses SSL/TLS encryption (port 587)
- ✅ Credentials in appsettings.json
- ✅ Should use environment variables in production
