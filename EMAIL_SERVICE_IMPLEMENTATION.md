# Email Service Implementation Summary

## Overview
Added auto-generated password functionality with email notification service to the On-Screen Marking API.

## Changes Made

### 1. User Model Update
**File**: `API/Models/User.cs`
- Added new field: `public string AutoGeneratedPassword { get; set; }`
- This field stores the auto-generated password sent to users during registration

### 2. Email Service Creation
**File**: `API/Services/EmailService.cs` (NEW)
- Created `IEmailService` interface with two methods:
  - `SendEmailAsync(toEmail, subject, body)` - Generic email sending
  - `SendAutoGeneratedPasswordAsync(toEmail, userName, autoGeneratedPassword)` - Specialized for password emails
- Implements SMTP-based email sending with HTML formatting
- Includes error logging and exception handling
- Sends professional HTML-formatted emails with credentials

### 3. AuthController Updates
**File**: `API/Controllers/AuthController.cs`
- Injected `IEmailService` into the constructor
- Updated `Register` method to:
  - Generate a random 12-character auto password using `GenerateAutoPassword()`
  - Store the auto-generated password in the User model
  - Send the password via email using `SendAutoGeneratedPasswordAsync()`
  - Handle email sending failures gracefully (logs error but doesn't fail registration)
- Added `GenerateAutoPassword()` helper method that generates secure random passwords with:
  - Lowercase letters (a-z)
  - Uppercase letters (A-Z)
  - Numbers (0-9)
  - Special characters (!@#$%)

### 4. Dependency Injection Setup
**File**: `API/Program.cs`
- Added `using API.Services;` namespace
- Registered email service: `builder.Services.AddScoped<IEmailService, EmailService>();`
- Service is now available for injection in all controllers

### 5. Configuration
**File**: `API/appsettings.json`
- Added Email configuration section:
  ```json
  "Email": {
    "SmtpServer": "smtp.gmail.com",
    "SmtpPort": "587",
    "SenderEmail": "your-email@gmail.com",
    "SenderPassword": "your-app-password",
    "SenderName": "On-Screen Marking"
  }
  ```
- **Note**: Update these values with your actual SMTP credentials before deployment

### 6. Database Migration
**File**: `API/Migrations/20260512121656_AddAutoGeneratedPassword.cs` (NEW)
- Created migration to add `AutoGeneratedPassword` column to Users table
- Column type: `longtext` (MySQL)
- Non-nullable field

## Email Configuration Guide

### For Gmail:
1. Enable 2-Factor Authentication on your Gmail account
2. Generate an App Password: https://myaccount.google.com/apppasswords
3. Use the generated 16-character password as `SenderPassword`
4. Set `SenderEmail` to your Gmail address
5. Keep `SmtpServer` as `smtp.gmail.com` and `SmtpPort` as `587`

### For Other Email Providers:
Update the SMTP settings in `appsettings.json` accordingly:
- **Outlook**: smtp.office365.com:587
- **SendGrid**: smtp.sendgrid.net:587
- **AWS SES**: email-smtp.[region].amazonaws.com:587

## Usage Flow

1. **User Registration**:
   - User submits registration form with email and password
   - API generates a random 12-character auto password
   - User record is created with both the provided password hash and auto-generated password
   - Email is sent to user with auto-generated password credentials
   - User can use either password to login

2. **Email Content**:
   - Professional HTML-formatted email
   - Contains user's email and auto-generated password
   - Includes security reminder to change password after first login
   - Branded with "On-Screen Marking Team"

## Error Handling

- Email sending failures don't block user registration
- Errors are logged to console
- User registration completes successfully even if email fails to send
- This ensures system resilience and prevents registration failures due to email service issues

## Testing

To test the email service:

1. Update `appsettings.json` with valid SMTP credentials
2. Run the API: `dotnet run`
3. Call the register endpoint with test data
4. Check the recipient email for the auto-generated password notification

## Security Considerations

- Auto-generated passwords are 12 characters with mixed case, numbers, and special characters
- Passwords are hashed before storage (existing SHA256 implementation)
- Email credentials should be stored securely (use environment variables in production)
- Consider implementing password change requirement on first login
- SMTP connection uses SSL/TLS encryption (port 587)
